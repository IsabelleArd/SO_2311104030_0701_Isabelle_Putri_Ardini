## Dasar Teori

Xinu (Xinu Is Not Unix) merupakan sistem operasi kecil yang dirancang untuk tujuan pembelajaran, khususnya dalam memahami konsep dasar sistem operasi seperti manajemen proses, komunikasi antar proses, serta pengelolaan memori. Xinu dikembangkan untuk digunakan pada lingkungan jaringan dan embedded system.

Pada praktikum ini digunakan metode cross development, yaitu proses pengembangan dilakukan pada satu sistem (development-system) sedangkan sistem operasi dijalankan pada sistem lain (backend). Dalam praktikum Xinu, kedua sistem tersebut dijalankan menggunakan Virtual Machine.

Development-system berfungsi sebagai mesin pengembang yang menjalankan sistem operasi Linux dan berisi source code Xinu, compiler, serta layanan jaringan seperti DHCP dan TFTP. Backend VM berfungsi sebagai mesin target yang akan menjalankan sistem operasi Xinu melalui proses booting jaringan menggunakan protokol PXE.

Melalui Xinu Shell, pengguna dapat berinteraksi langsung dengan sistem operasi menggunakan berbagai perintah. Shell ini memungkinkan pengguna untuk melihat proses yang sedang berjalan, konfigurasi jaringan, serta menjalankan utilitas sistem lainnya.

### 2. Menjalankan Xinu
Langkah-langkah menjalankan Xinu:

1.  Menjalankan VirtualBox kemudian Start
    **2311104030-development-system**.
2.  Login menggunakan akun:
    -   Username: **xinu**
    -   Password: **xinurocks**
3.  Membuka terminal kemudian masuk ke direktori compile:

```
    cd xinu/compile
```

4.  Menghapus hasil kompilasi sebelumnya dengan perintah:

```
    make clean
```

5.  Mengompilasi kembali source code Xinu dengan perintah:

```
    make
```

6.  Menjalankan backend VM.

7.  Menjalankan **minicom** untuk menghubungkan development-system
    dengan backend:

```
    sudo minicom
```

Jika berhasil, sistem akan menampilkan **Xinu Shell** dengan prompt:

    xsh$

Beberapa perintah yang tersedia pada Xinu Shell beserta fungsinya antara
lain:

| No | Perintah       | Fungsi                                                                 |
|----|----------------|------------------------------------------------------------------------|
| 1  | `argecho`      | Menampilkan kembali argumen yang diberikan oleh pengguna               |
| 2  | `cat`          | Menampilkan isi dari sebuah file                                       |

| 4  | `date`         | Menampilkan tanggal dan waktu saat ini                                 |
| 5  | `devdump`      | Menampilkan informasi tentang device yang terdaftar pada sistem        |
| 6  | `echo`         | Menampilkan teks atau string yang diberikan ke layar                   |
| 7  | `exit`         | Keluar dari shell Xinu                                                 |
| 8  | `help`         | Menampilkan daftar semua perintah yang tersedia pada Shell Xinu        |
| 9  | `kill`         | Menghentikan (terminate) sebuah proses berdasarkan PID-nya             |

### 3. Pertanyaan
1.  **Berapa jumlah perintah pada Xinu?**\
    Jumlah perintah pada Xinu adalah sekitar **20 perintah** (tergantung
    versi Xinu yang digunakan).

2.  **Sebutkan 2 perintah yang mempunyai fungsi yang sama!**\
    Contohnya:\
    `exit` dan `logout` yang sama-sama digunakan untuk keluar dari
    shell.

3.  **Berapa IP address Xinu?**

```
    192.168.1.2
```

4.  **Perintah apa yang digunakan untuk mengetahui IP address?**

```
    ifconfig
```

atau

    netstat

5.  **Berapa IP DNS server yang digunakan oleh Xinu?**

```
    192.168.1.1
```

6.  **Terdapat berapa proses yang sedang berjalan pada Xinu?**\
    Biasanya terdapat sekitar **10 proses** yang berjalan secara
    default.

7.  **Proses apa yang mempunyai prioritas paling rendah?**\
    Proses dengan prioritas paling rendah biasanya adalah **null
    process**.

8.  **Proses apa yang mempunyai ukuran paling besar?**\
    Proses dengan ukuran paling besar biasanya adalah **main process**.

9.  **Proses apa yang berada dalam state current?**\
    Proses yang berada dalam state current adalah **main process**.

10. **Proses apa yang berada dalam state suspend?**\
    Proses suspend biasanya merupakan proses yang sedang menunggu untuk
    dijalankan kembali.

11. **Berapa PID (Process ID) dari Main process?**

```
    PID = 1
```

# Catatan Xinu

### Cara keluar dari Minicom

1.  Tekan **Ctrl + A**
2.  Lepaskan tombol
3.  Tekan **X**
4.  Pilih **Yes**

### Jika Xinu mengalami masalah (hang)

1.  Matikan **Backend VM**
2.  Keluar dari Minicom
3.  Jalankan kembali:

```
    sudo minicom
```

4.  Nyalakan kembali Backend VM
5.  Jika masih bermasalah, restart development-system
